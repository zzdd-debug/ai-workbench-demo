# Demo Script

## 适用场景
- 面试演示
- 录屏讲解
- 项目答辩
- 本地方案走查

## 演示目标
在 5 到 8 分钟内清晰展示这套 Demo 的价值、主链路、能力边界和可扩展方向。

## 演示前准备
1. 启动后端
   ```powershell
   Set-Location C:\ai-rd-workbench-demo\backend
   $env:JAVA_HOME='D:\Java'
   $env:PATH='D:\Java\bin;' + $env:PATH
   .\mvnw.cmd spring-boot:run
   ```
2. 启动前端
   ```powershell
   Set-Location C:\ai-rd-workbench-demo\frontend
   npm run dev
   ```
3. 打开浏览器访问 `http://127.0.0.1:5173`
4. 准备演示账号：`demo / 123456`

## 推荐讲解顺序
### 1. 项目定位
- 这是一个面向开发交付的 AI Agent Workflow Demo
- 第一版默认运行在 MockProvider 下，确保演示稳定
- 重点不是炫技，而是把研发工作台的最小主链路闭环跑通

### 2. 登录入口
- 先展示未登录访问工作台会进入登录页
- 输入 `demo / 123456`
- 点击登录后进入工作台
- 补一句：这里是演示级 auth，不做复杂鉴权体系

### 3. 自然语言任务输入
- 在输入区输入一句自然语言任务
- 推荐示例：
  - `请帮我拆解一个 AI 研发任务工作台 Demo 的交付任务`
- 点击提交，说明这里会创建任务并触发 mock agent 解析

### 4. 看板展示
- 展示任务被拆成多个步骤卡片
- 强调第一版采用列式分栏：
  - 待开始
  - 进行中
  - 待验证
  - 已完成
- 说明顶层任务状态不是手动改，而是由步骤状态汇总

### 5. 日志与知识引用
- 切到执行日志区
- 说明日志类型至少区分：
  - `knowledge`
  - `decision`
  - `tool`
  - `error`
- 再展示知识引用区
- 强调第一版知识源是静态文档，不做在线编辑和动态写回

### 6. 步骤状态更新与摘要联动
- 手动把某个步骤状态从 `NOT_STARTED` 改为 `IN_PROGRESS` 或 `COMPLETED`
- 观察看板列变化
- 观察任务级摘要和下一步建议同步变化
- 强调摘要只保留任务级，不扩展步骤级摘要

### 7. 边界说明
- 当前版本不接真实模型
- 不接数据库
- 不接真实 RAG / MCP
- 适合演示主链路、系统边界和后续扩展思路

## 推荐演示口径
- “我优先保证这套 Demo 稳定可启动、可演示，而不是一开始就引入真实模型和复杂基础设施。”
- “日志、知识引用、摘要和步骤状态都被独立展示出来，方便把 Agent 行为讲清楚。”
- “这套结构后续可以继续接 RAG、MCP 或真实 provider，但第一版先把交付闭环做扎实。”

## 录屏版简化脚本
1. 打开登录页并说明项目定位
2. 登录进入工作台
3. 输入自然语言任务并生成步骤
4. 展示看板、日志、知识引用
5. 手动更新步骤状态并展示摘要联动
6. 用一句话收尾：
   - “这套 Demo 已经完成最小闭环，下一步更适合补齐发布材料和扩展位，而不是继续堆功能。”

## 常见问答提示
- 为什么不用真实模型？
  - 为了降低演示成本和环境不确定性，先保证链路可演示、可说明。
- 为什么不接数据库？
  - 第一版目标是工作流骨架验证，不是数据基础设施建设。
- 这套系统后续能扩展什么？
  - 真实 provider、RAG、MCP、细粒度 Agent 运行详情、更多任务协作能力。
